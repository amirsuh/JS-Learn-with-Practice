A closure is a function that remembers variables from its lexical scope, even after the outer function has finished executing. Lexical scope means the scope of a variable is determined by its position in the code. An inner function can access variables from its outer function and even global scope. Closures are what allow functions to “carry” their environment with them.

Hoisting - Full Explanation

Hoisting is a default behavior of JS where variables and function declarations are moved to the top of their scope before code execution. Variables with var are initialized as undefined. Functions can be called before they are written. let and const are hoisted but in TDZ until initialized.

Hoisting - Key Points

  • Declarations are moved to the top of their scope.
  • var variables hoisted as undefined.
  • Function declarations fully hoisted.
  • let and const stay in TDZ until initialized.

Closures - Explanation

A closure is a function that remembers variables from its lexical scope, even after the outer function has finished executing. Inner functions can access outer function variables and global scope.

Closures - Key Points

  • Lexical scope is determined by code position.
  • Inner functions can access outer and global variables.
  • Closures “carry” their environment with them.
