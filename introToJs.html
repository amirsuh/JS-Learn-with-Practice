JavaScript Execution in the Browser

JavaScript runs via the browser's JS Engine

  • v8 in Chrome
  • SpiderMonkey in Firefox
  • JavaScriptCore in Safari
  • Chakra in Edge (now v8)
Execution phases
  • Parsing → converts code to AST (Abstract Syntax Tree)
  • Compilation
  • Execution

An AST represents the code's structure in a tree format, helping compilation and execution.

let x = 5; → tokens: let, x, =, 5, ;
JavaScript Execution Process
  1. Parsing: Code → AST
  2. Compilation: AST → Bytecode → Optimized Machine Code
  3. Execution: CPU runs machine code; GC cleans unused memory
Garbage Collection in JS

JS automatically frees memory of unused objects using "Mark-and-Sweep". Reference counting exists but is rarely used due to cyclic reference issues.

  1. Mark Phase: Find reachable objects from global/function scope
  2. Sweep Phase: Remove unreachable objects
function reachable() {
let object = { a: 1 }; // Reachable
} // object unreachable after function ends
