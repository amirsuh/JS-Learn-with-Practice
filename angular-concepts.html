Angular Core Concepts

Angular Core

Angular core provides foundational APIs like @Component, @NgModule, @Injectable, and CommonModule for bootstrapping apps. The root module (AppModule) declares components and imports essential modules. Core handles change detection via Zone.js, tracking async operations automatically.

        
          import { NgModule } from '@angular/core';
          
          @NgModule({
            declarations: [AppComponent],
            imports: [BrowserModule],
            bootstrap: [AppComponent]
          })
          export class AppModule { }

Components

Components encapsulate template, styles, and logic using @Component decorator with selector, templateUrl, and styleUrls. They implement lifecycle hooks like ngOnInit() for initialization. Components communicate via @Input() for data down and @Output() for events up.

        
          import { Component } from '@angular/core';

          @Component({
            selector: 'app-user',
            template: '<h1>{{name}}</h1>',
            styleUrls: ['./user.component.css']
          })
          export class UserComponent {
            @Input() name: string = '';
          }

Modules

Modules (@NgModule) group related components, directives, and services. Feature modules organize app sections; root module bootstraps the app. Shared modules export reusable elements for multiple features.

        
          @NgModule({
            declarations: [UserComponent],
            imports: [CommonModule],
            exports: [UserComponent]  // For shared modules
          })
          export class UserModule { }

Templates

Templates use HTML with Angular syntax: interpolation {{expression}}, property binding [prop]="value", event binding (event)="handler()". Structural directives like *ngIf, *ngFor control rendering. Property binding updates DOM attributes dynamically.

        
          <!-- user.component.html -->
          <div *ngIf="user">
            <p>{{user.name}}</p>
            <button (click)="onSave()">Save</button>
          </div>
          <input [value]="user.email" />

Forms

Template-Driven Forms

Template-driven forms use ngModel, ngForm directives. Two-way binding with [(ngModel)]. Access form via #form="ngForm". Suitable for simple forms.

        
          <form #userForm="ngForm" (ngSubmit)="onSubmit(userForm)">
            <input name="name" ngModel required />
            <button [disabled]="!userForm.valid">Submit</button>
          </form>

Reactive Forms

Reactive forms use FormControl, FormGroup, FormArray from ReactiveFormsModule. Programmatic validation/control. Immutable, testable structure.

        
          import { FormBuilder, Validators } from '@angular/forms';

          ngOnInit() {
            this.userForm = this.fb.group({
              name: ['', [Validators.required, Validators.minLength(3)]]
            });
          }
        
      
        
          <form [formGroup]="userForm" (ngSubmit)="onSubmit()">
            <input formControlName="name" />
            <div *ngIf="userForm.get('name')?.invalid">Invalid</div>
          </form>

HTTP and Services

HttpClient

HttpClient (requires HttpClientModule) returns Observables. Supports GET, POST, PUT, DELETE with typing via generics. Configure base URL in interceptors. Use HttpParams for query strings.

        
          import { HttpClient } from '@angular/common/http';

          constructor(private http: HttpClient) { }

          getUsers() {
            return this.http.get<User[]>('/api/users');
          }

          postUser(user: User) {
            return this.http.post<User>('/api/users', user);
          }

HTTP Interceptors

Interceptors (@Injectable()) hook into HTTP requests/responses app-wide. Add auth headers, log requests, handle errors. Implement HttpInterceptor interface, provide in HTTP_INTERCEPTORS.

        
          import { HttpInterceptorFn } from '@angular/common/http';

          export const authInterceptor: HttpInterceptorFn = (req, next) => {
            const authReq = req.clone({
              headers: req.headers.set('Authorization', 'Bearer ' + token)
            });
            return next(authReq);
          };

Routing

RouterModule.forRoot() configures routes with path, component, children. RouterLink directive for navigation. ActivatedRoute accesses params/query. Guards protect routes.

        
          // app-routing.module.ts
          const routes: Routes = [
            { path: 'users/:id', component: UserDetailComponent },
            { path: '**', component: NotFoundComponent }
          ];

Lazy Loading

Load feature modules on-demand with loadChildren. Reduces initial bundle size. Use loadChildren: () => import('./module').then(m => m.Module).

        
          { path: 'admin', loadChildren: () => 
            import('./admin/admin.module').then(m => m.AdminModule) }

State Management (NgRx)

NgRx implements Redux pattern: Actions → Reducers → Store → Selectors → Effects (async). Store holds immutable state. Effects handle side effects like API calls.

        
          // action
          export const loadUsers = createAction('[User] Load');

          // reducer
          on(loadUsers, (state) => ({ ...state, loading: true }));

Conclusion

Angular's architecture supports building scalable, maintainable single-page applications. By mastering core concepts like components, services, routing, HTTP, and state management, you can create powerful and performant apps with Angular.
